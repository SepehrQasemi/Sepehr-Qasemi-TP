Projet final CRM Food Trading

Conception et livraison d un CRM SaaS pour une entreprise de commerce B2B en matieres premieres alimentaires.

Scenario entreprise

Contexte reel utilise pour la demonstration

L entreprise traite des demandes clients sur des produits comme cacao, poudre de lait et melanges d epices. Le cycle commercial demande une qualification stricte, des devis suivis, puis une relance automatique si le prospect reste inactif. Le CRM centralise les contacts, les entreprises, les leads, les taches et les emails pour fiabiliser ce processus.

Workflow et pipeline dedies

Pipeline vente adapte au metier

Pipeline

  • Nouveau lead
  • Qualification
  • Echantillon envoye
  • Devis envoye
  • Negociation
  • Gagne
  • Perdu

Le workflow de suivi inclut creation de taches, changement de stage, historisation, et relance automatique a 72h sur le stage "Devis envoye".

Outils utilises

Stack technique et delivery
  • Next.js 16 (frontend + API routes)
  • Supabase (PostgreSQL, Auth, RLS)
  • Brevo (emails transactionnels)
  • Vercel (deployment)
  • GitHub + GitHub Actions (versioning + CI)
